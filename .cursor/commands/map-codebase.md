# Map Codebase

Analyze the existing codebase and produce structured documentation in `.planning/codebase/`.

Spawns 4 analysis passes (tech, architecture, quality, concerns) and writes 7 documents:
STACK.md, INTEGRATIONS.md, ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md, CONCERNS.md.

Use the **map-codebase** skill for the full workflow, templates, and document quality rules.

**When to run:**
- Before starting work on a brownfield project (understand existing code first)
- To refresh documentation after major changes
- When onboarding to an unfamiliar codebase

**Output:** `.planning/codebase/` with 7 committed documents ready for future planning and coding sessions.
