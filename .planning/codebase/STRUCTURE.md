# Codebase Structure

**Analysis Date:** 2026-02-09

## Directory Layout

```
stranded/
├── src/                  # Source code implementations
│   ├── actors/          # Game entity implementations
│   ├── core/            # Core game systems
│   └── main.cpp         # Application entry point
├── include/             # Header files and interfaces
├── graphics/            # Game assets (BMP + JSON descriptors)
│   ├── bg/              # Background tiles and maps
│   └── sprite/          # Sprite graphics by category
├── audio/               # Music and sound effects
├── dmg_audio/           # Game Boy audio format files
├── butano/              # Butano engine submodule
├── tools/               # Development and utility tools
├── scripts/             # Build and launch scripts
└── Makefile             # Build configuration
```

## Directory Purposes

**src/:**
- Purpose: C++ implementation files
- Contains: Game logic, entity behaviors, scene implementations
- Key files: `src/main.cpp` (entry point), `src/core/world.cpp` (main game loop)

**include/:**
- Purpose: C++ header files and class declarations
- Contains: Interface definitions, constants, forward declarations
- Key files: `include/str_player.h` (main player class), `include/str_scene.h` (scene enum)

**graphics/:**
- Purpose: Visual assets in GBA-compatible formats
- Contains: BMP images with JSON descriptors generated by Grit
- Key files: Background tiles, sprite sheets, UI elements

**audio/:**
- Purpose: Music and sound files for Maxmod
- Contains: MOD/XM/S3M/IT tracker files
- Key files: `audio/crystal.it`, `audio/death.wav`

**butano/:**
- Purpose: Butano engine library as git submodule
- Contains: Complete GBA development framework
- Key files: Engine source, examples, tools

## Key File Locations

**Entry Points:**
- `src/main.cpp`: Main application loop and scene management
- `Makefile`: Build system configuration and asset paths

**Configuration:**
- `include/str_constants.h`: Game constants and configuration values
- `Makefile`: Build flags, source paths, asset compilation

**Core Logic:**
- `src/core/world.cpp`: Main gameplay implementation
- `src/actors/player.cpp`: Player character logic
- `include/str_scene*.h`: Scene class definitions

**Assets:**
- `graphics/sprite/player/`: Player character graphics
- `graphics/bg/`: Background tile sets
- `audio/`: Music and sound effects

## Naming Conventions

**Files:** snake_case.cpp / snake_case.h (e.g., `player.cpp`, `str_scene.h`)
**Directories:** snake_case (e.g., `graphics/`, `include/`)
**Classes/Types:** PascalCase (e.g., `Player`, `WorldState`)
**Functions:** snake_case() (e.g., `execute()`, `update()`)
**Constants:** UPPER_SNAKE_CASE (e.g., `MAIN_WORLD_SPAWN_X`)

## Where to Add New Code

**New feature:**
- Primary code: `src/core/` for systems, `src/actors/` for entities
- Tests: No automated test framework - test in emulator

**New module/component:**
- Implementation: `src/core/[module_name].cpp`
- Header/interface: `include/str_[module_name].h`

**New asset:**
- Location: `graphics/sprite/[category]/` or `graphics/bg/`
- Descriptor: JSON file auto-generated by build process

**Utilities:**
- Shared helpers: `src/core/` with header in `include/str_[utility].h`

## Special Directories

**butano/:**
- Purpose: Butano engine library
- Generated: No - git submodule
- Committed: Yes - tracked via git submodule

**build/ (generated):**
- Purpose: Build artifacts and object files
- Generated: Yes - created by Make build system
- Committed: No - excluded by .gitignore

**tools/mGBA-0.10.5-win64/:**
- Purpose: Development emulator
- Generated: No - distributed with project
- Committed: Yes - for consistent development environment
